# Markdown 排版优化 Skill

## 文章结构规范

### 1. Front Matter (头部元数据)

```markdown
---
title: 文章标题（中文，清晰明了）
date: YYYY-MM-DD HH:mm:ss
tags:
  - 标签1
  - 标签2
  - 标签3
categories:
  - 分类名
---
```

### 2. 开头概述

在正文开头添加简短概述，告诉读者这篇文章讲什么：

```markdown
## 概述

本文是关于 XXX 的学习记录，记录了 XXX 的核心概念和实现思路。

---
```

## 章节排版规范

### 章节标题

使用二级标题 `##` 作为主要章节标题，包含章节名和简短描述：

```markdown
## Chapter 1 - Agent Loop

## 第一章 - 入门指南
```

### 章节分隔

每个主要章节之间用分隔线分开：

```markdown
---
```

## 内容排版规范

### 1. 重点强调

使用粗体强调核心概念：

```markdown
**核心思想：** 渐进式披露
**关键设计：** Subagent 实现了会话隔离
```

### 2. 列表使用

- 使用无序列表整理要点（`-` 符号）
- 逻辑清晰的条目适合用列表展示
- 列表项之间保持适当间距

### 3. 引用内容

使用块引用展示重要思考或引用：

```markdown
> "干净回滚"指的是能够可靠地撤销某个 Agent 做的所有改动，而不影响其他 Agent 的工作。
```

### 4. 代码片段

短代码使用行内代码格式，长代码使用代码块并指定语言：

行内代码：
```markdown
调用 `agent_loop` 函数处理消息
```

代码块：
```python
with open(inbox_path, "a") as f:
    f.write(json.dumps(msg) + "\n")
```

### 5. 表格对比

对比类内容使用表格展示：

```markdown
| 特性 | Worktree | Git Branch |
|------|----------|------------|
| 目录结构 | 物理复制多份目录 | 同一目录切换分支 |
| 适用场景 | 多 Agent 并行操作 | 单线程任务 |
```

## 标题层级规范

```markdown
## 二级标题 - 主要章节

### 三级标题 - 小节标题

#### 四级标题 - 子项标题
```

- 不要在文章中使用 `#` 一级标题（由主题自动生成）
- 二级标题用于主要章节
- 三级标题用于小节
- 四级标题用于子项

## 可读性优化

### 1. 段落拆分

- 长段落拆分为短段落，每段聚焦一个要点
- 段落之间空一行
- 每段不超过 3-4 行

### 2. 结构化展示

```markdown
**工作流程：**
1. 第一步...
2. 第二步...
3. 第三步...
```

### 3. 概念解释

对于核心概念，单独抽出解释：

```markdown
**核心机制：**
- 后台任务以子进程方式启动
- 使用 `with self._lock` 自动加锁解锁
```

## 示例效果

### ❌ 优化前

```markdown
# lcc chapter8 Background Tasks
本节添加了后台任务管理系统，通过工具调用可以实现后台任务启动，即llm可以根据用户提问来判断是否调用工具，调用之后后台任务就会以子进程方式启动。然后在agent_loop内的每次对话循环中，都会先排空一遍消息队列。这个操作是在后台子进程中完成，排空（drain）操作，使用with self._lock，这样可以自动加锁解锁。
```

### ✅ 优化后

```markdown
## Chapter 8 - Background Tasks

后台任务管理系统，通过工具调用启动后台任务。

**核心机制：**
- 后台任务以子进程方式启动
- `agent_loop` 每次对话前，先排空一遍消息队列
- 使用 `with self._lock` 自动加锁解锁
- 子进程执行期间，排空操作会等待
- 执行完成后解锁，进行排空
```
